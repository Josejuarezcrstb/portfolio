## Supply Chain Analytics Architecture

This case study outlines the development of an interactive dashboard for Supply Chain leaders at a leading Mexican company. The project's goal is to provide real-time visibility to balance the workload among buyers and monitor key performance indicators (KPIs).

The system was built with Blazor WebAssembly (frontend), Appwrite (authentication and backend), and Azure SQL Database (data layer). The solution targets a large-scale corporate environment, utilizing pre-normalized data from a global ERP system.

---

## The Challenge

- Lack of real-time operational visibility for purchasing coordinators.
- Requirement for a secure and fast interface with sub-2-second latency.
- Integration within a distributed architecture crossing vendor boundaries (Blazor + Appwrite + Azure SQL) and strict role-based access control (coordinator/buyer/admin).
- Need to generate workload saturation alerts and actionable data without modifying the transactional database.

## The Architecture

- Frontend: Blazor WebAssembly (SPA) featuring chart controls, dynamic filters, and KPI cards.
- Authentication: Appwrite Cloud with JWT and role management.
- API: ASP.NET Core Minimal API for token validation and parameterized SQL queries.
- Database: Azure SQL with normalized relational tables (SAP S/4HANA-style).
- Strategic Endpoints: `/api/workload/kpis`, `/api/workload/buyer-load`, and date range filters.

## Advanced Troubleshooting

During the technical integration, I resolved multiple infrastructure and network challenges:

1. CORS and Origin Limits
   - Adjusted CORS rules within Appwrite and configured appropriate headers in Blazor.
   - Validated that preflight requests (OPTIONS) resolved with a 204 status and that Authorization and Content-Type headers were permitted.

2. Authentication and Token Flow
   - Implemented middleware in the Minimal API to verify Appwrite JWT validity and extract user roles.
   - Diagnosed 401/403 errors related to expired tokens or insufficient role permissions.

3. Azure SQL and Throttling
   - Tuned the Azure SQL connection with retry policies and scalable timeouts.
   - Monitored aggregation queries (SUM, COUNT, GROUP BY) to prevent row-lock blocking.

4. GitHub Copilot CLI for Advanced Debugging
   - Utilized Copilot CLI to inject real-time diagnostics, including stack trace streaming, container inspection commands, and Appwrite header analysis.
   - Transitioned from a "guess and refresh" approach to a methodical, testable process, yielding reproducible results for 409 conflicts and adjustments in blazor.webassembly.js.

## Impact & Next Steps

**Early Results:**
- Fully functional dashboard displaying workload KPIs per buyer.
- Traffic-light mechanism (red/yellow/green) for proactive task reassignment.
- Dataset export capabilities to CSV/Excel for reporting.

**Next Steps:**
- Advance caching and query prefetching strategies to consistently maintain < 2s latency.
- Refine visualization UX with smooth animations and role-specific KPI pages.
- Incorporate automated recommendations through business rules (and/or lightweight ML models) based on historical workload data.

This project validates the ability to design and operate secure, scalable, and business-oriented solutions in distributed systems, maintaining a strong focus on data quality and response times.
